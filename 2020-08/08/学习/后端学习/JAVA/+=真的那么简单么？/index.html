+=你不知道的事儿

看一道面试题：

总结：

结果是第一个代码有问题
那为什么第二个木有问题呢?
    扩展的赋值运算符其实隐含了一个强制类型转换。
    s += 1;
    不是等价于 s = s + 1;
    而是等价于 s = (s的数据类型)(s + 1);

其实可以发现，平时用类型省时省力用这+=，熟不知它还有一个类型的强制转换
